# Compliance

Certification and compliance artifacts for DO-178C, DO-297, DO-326A/356A standards.

## Standards Coverage

### DO-178C — Software Considerations in Airborne Systems and Equipment Certification
Lifecycle artifacts for software development and verification:
- Plans (PSAC, SDP, SVP, SCMP, SQA)
- Standards (coding, design, requirements)
- Software Requirements Data
- Software Design Description
- Test cases and procedures
- Verification results
- Configuration management records

### DO-297 — Integrated Modular Avionics (IMA) Development Guidance
IMA-specific objectives:
- Partition configuration management
- APEX interface specifications
- Resource allocation verification
- Partition isolation evidence
- Schedule feasibility analysis
- Health monitoring verification

### DO-326A — Airworthiness Security Process Specification
Security development process:
- Security risk assessment
- Threat analysis
- Security objectives
- Security assurance
- Security verification

### DO-356A — Airworthiness Security Methods and Considerations
Security methods application:
- Penetration testing
- Vulnerability assessment
- Security architecture review
- Cryptographic implementation
- Access control verification

## Evidence Organization

Artifacts organized by:
- Standard and objective
- Development phase
- Verification activity
- Compliance status

Links to:
